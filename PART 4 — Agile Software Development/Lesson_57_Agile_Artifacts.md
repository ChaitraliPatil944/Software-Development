# Introduction

Agile ceremonies organize **communication**.

Agile artifacts organize **work**.

Without artifacts, teams would struggle to track priorities, sprint goals, and completed work.

```text
Ideas
  ↓
Product Backlog
  ↓
Sprint Backlog
  ↓
Product Increment
```

# What are Agile Artifacts?

Artifacts are important pieces of information that make work visible and transparent.

The three official Scrum artifacts are:

- Product Backlog
- Sprint Backlog
- Product Increment

---

# Overall Flow

```text

      ↓
Sprint Backlog

      ↓

```

# Artifact 1 — Product Backlog

## What is it?

An ordered list of everything that might be needed in the product.

It changes continuously as business needs evolve.

### Owner

**Product Owner**

### Contents

- User Stories
- Bug Fixes
- Technical Improvements
- Research Tasks
- Security Enhancements

### Example

```text
Product Backlog

1. User Login
2. Search Products
3. Shopping Cart
4. Payments
5. Notifications
```

---

# Product Backlog Characteristics

- Prioritized
- Dynamic
- Continuously refined
- Visible to the team
- Ordered by business value

---

# Artifact 2 — Sprint Backlog

## What is it?

The Sprint Backlog contains the work selected for the current sprint.

It answers:

- What will we build?
- How will we build it?

### Owner

**Developers**

### Created During

Sprint Planning

### Example

```text
Sprint Goal:
Implement Shopping Cart

Sprint Backlog

✓ Create Cart API
✓ Build UI
✓ Write Tests
✓ Review Code
```

---

# Sprint Backlog Characteristics

- Sprint-specific
- Updated daily
- Supports the Sprint Goal
- Managed by the development team

---

# Artifact 3 — Product Increment

## What is it?

The Product Increment is the completed, tested, integrated work produced during the sprint.

It must satisfy the team's **Definition of Done**.

### Example

Sprint Result:

```text
✓ Login
✓ Registration
✓ Cart

Ready for Release
```

---

# Definition of Done (DoD)

The Definition of Done ensures quality.

Typical checklist:

- Code completed
- Code reviewed
- Unit tests passed
- Integration tests passed
- Documentation updated
- Ready for deployment

Only then is work considered complete.

# Common Mistakes

❌ Treating the Product Backlog as a fixed document.

✔ Update it continuously.

❌ Adding random work to the Sprint Backlog.

✔ Protect the Sprint Goal.

❌ Calling incomplete work an increment.

✔ Only completed work becomes part of the Product Increment.

# Summary

Scrum artifacts provide transparency and structure throughout Agile development. The Product Backlog captures everything the product may need, the Sprint Backlog defines the work for the current sprint, and the Product Increment represents completed, high-quality software. Together, these artifacts help teams organize priorities, track progress, and deliver customer value in every sprint.
