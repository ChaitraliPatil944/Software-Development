# Introduction

Every software project has limits.

A project cannot build everything requested by everyone. Humans
naturally add "just one more feature" until a small application becomes
a monster with a calendar, payment gateway, AI assistant, and probably a
toaster integration. Scope Management exists to prevent that chaos.

``` text
Project Goal

      ↓

Define Scope

      ↓

Control Changes

      ↓

Successful Delivery
```

------------------------------------------------------------------------

# What is Project Scope?

**Project Scope** defines the boundaries of a project.

It describes:

-   What work will be completed
-   What features will be delivered
-   What requirements are included
-   What is outside the project

------------------------------------------------------------------------

# Why Do We Need Scope Management?

Without scope management:

``` text
Uncontrolled Requirements

        ↓

Extra Work

        ↓

More Cost

        ↓

Delayed Delivery
```

Scope Management helps:

-   Maintain project direction
-   Control workload
-   Manage expectations
-   Prevent unnecessary changes
-   Improve project success

------------------------------------------------------------------------

# What is Scope Management?

**Scope Management** is the process of defining, controlling, and
managing all work required to complete a project successfully.

It includes:

``` text
Scope Management

        |
-------------------------
|        |        |      |
Plan   Define  Verify  Control
```

# 1. Scope Planning

Scope planning defines how project scope will be managed.

It answers:

-   How will requirements be collected?
-   How will changes be handled?
-   Who approves changes?

------------------------------------------------------------------------

# 2. Scope Definition

Scope definition converts requirements into clear project boundaries.

Example:

## Food Delivery Application

Included:

    User Registration

    Restaurant Search

    Food Ordering

    Payment

Not Included:

    Food Preparation Management

------------------------------------------------------------------------

# 3. Requirement Collection

Requirements are gathered from stakeholders.

Sources:

-   Customers
-   Users
-   Business teams
-   Developers

Techniques:

-   Interviews
-   Surveys
-   Meetings
-   Prototypes

------------------------------------------------------------------------

# 4. Work Breakdown Structure (WBS)

WBS divides project scope into smaller tasks.

``` text
Project

   ↓

Modules

   ↓

Tasks

   ↓

Activities
```

Example:

    Online Banking System

    ├── Login Module

    ├── Account Module

    ├── Transaction Module

    └── Testing

------------------------------------------------------------------------

# 5. Scope Verification

Scope verification ensures completed work matches requirements.

Activities:

-   Review deliverables
-   Get stakeholder approval
-   Confirm requirements

Example:

Customer verifies:

    Payment Feature Completed

------------------------------------------------------------------------

# 6. Scope Control

Scope control manages changes during the project.

It involves:

-   Tracking changes
-   Evaluating impact
-   Approving or rejecting requests

------------------------------------------------------------------------

# Scope Creep

## What is Scope Creep?

Scope creep is uncontrolled expansion of project requirements without
proper approval.

Example:

Initial requirement:

    Build a website

Later requests:

    Add Mobile App

    Add Chat System

    Add AI Assistant

    Add Analytics

The project grows beyond original limits.

------------------------------------------------------------------------

# Causes of Scope Creep

Common causes:

-   Poor requirement analysis
-   Unclear objectives
-   Frequent customer changes
-   Lack of change process
-   Poor communication

------------------------------------------------------------------------

# Effects of Scope Creep

Scope creep causes:

-   Increased cost
-   Delayed deadlines
-   Resource problems
-   Reduced quality
-   Team stress

------------------------------------------------------------------------

       ↓

Approval

------------------------------------------------------------------------

# Change Control Board (CCB)

A Change Control Board reviews major project changes.

Responsibilities:

-   Analyze requests
-   Evaluate impact
-   Approve/reject changes

------------------------------------------------------------------------

# Scope Statement

A scope statement documents:

-   Project objectives
-   Deliverables
-   Features
-   Limitations
-   Assumptions

Example:

    Project:
    Online Learning Platform

    Deliverables:
    Course Management
    Online Exams
    Student Dashboard

------------------------------------------------------------------------

# Scope Management Example

## Hospital Management System

Scope:

Included:

    Patient Registration

    Appointment Booking

    Medical Records

Excluded:

    Insurance Processing

A new insurance feature requires change approval.

------------------------------------------------------------------------

# Scope Management in Agile

Agile handles scope differently.

Instead of fixing everything upfront:

``` text
Product Backlog

        ↓

Prioritize Features

        ↓

Sprint Development

        ↓

Continuous Changes
```

Changes are managed through backlog prioritization.

------------------------------------------------------------------------

# Scope Management vs Requirement Management

  Scope Management              Requirement Management
  ----------------------------- -------------------------------
  Controls project boundaries   Manages detailed needs
  Defines what is included      Defines what system should do
  Focuses on project work       Focuses on features

------------------------------------------------------------------------

# Common Mistakes

## Mistake 1

Starting development without clear scope.

Solution:

Define boundaries first.

------------------------------------------------------------------------

## Mistake 2

Accepting every change immediately.

Solution:

Analyze impact before approval.

------------------------------------------------------------------------

## Mistake 3

Ignoring stakeholders.

Solution:

Maintain regular communication.

------------------------------------------------------------------------

# Best Practices

-   Clearly document scope
-   Define exclusions
-   Use WBS
-   Manage change requests
-   Communicate with stakeholders
-   Review scope regularly

# Summary

Scope Management ensures that software projects deliver the expected
features within defined boundaries. It helps teams control requirements,
prevent scope creep, manage changes, and maintain project focus.
Effective scope management improves cost control, scheduling, and
overall project success.
