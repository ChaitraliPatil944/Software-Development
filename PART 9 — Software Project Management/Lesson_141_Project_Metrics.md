# Introduction

Software projects cannot be managed effectively without measurement.

A project manager needs answers:

-   Are we on schedule?
-   Is quality improving?
-   Are we within budget?
-   Is the team productive?

Project Metrics provide measurable information about project
performance.

``` text
Project Activities

        ↓

Measurement

        ↓

Analysis

        ↓

Improvement
```

------------------------------------------------------------------------

# What are Project Metrics?

**Project Metrics** are quantitative measurements used to evaluate the
performance, progress, quality, and health of a software project.

They help measure:

-   Time
-   Cost
-   Quality
-   Productivity
-   Risks
-   Team performance

------------------------------------------------------------------------

# Why Do We Need Project Metrics?

Without metrics:

``` text
No Measurement

      ↓

No Visibility

      ↓

Poor Decisions

      ↓

Project Problems
```

Metrics help:

-   Track progress
-   Identify problems early
-   Improve decision making
-   Control resources
-   Predict outcomes

# 1. Schedule Metrics

Measure project timeline performance.

Examples:

-   Task completion rate
-   Schedule variance
-   Milestone completion

------------------------------------------------------------------------

# Schedule Variance

Formula:

    Schedule Variance

    =

    Actual Progress

    -

    Planned Progress

Example:

Planned:

    50% completion

Actual:

    40% completion

Variance:

    -10%

Project is behind schedule.

------------------------------------------------------------------------

# 2. Cost Metrics

Measure financial performance.

Examples:

-   Budget usage
-   Cost variance
-   Actual spending

------------------------------------------------------------------------

# Cost Variance

Formula:

    Cost Variance

    =

    Budgeted Cost

    -

    Actual Cost

Example:

Budget:

    ₹10 Lakhs

Spent:

    ₹12 Lakhs

Variance:

    -₹2 Lakhs

Over budget.

------------------------------------------------------------------------

# 3. Quality Metrics

Measure software quality.

Examples:

-   Defect count
-   Defect density
-   Test coverage
-   Bug resolution time

------------------------------------------------------------------------

# Defect Density

Formula:

    Defect Density

    =

    Number of Defects

    /

    Size of Software

Example:

    20 bugs

    /

    1000 lines of code

------------------------------------------------------------------------

# 4. Productivity Metrics

Measure team efficiency.

Examples:

-   Lines of code
-   Features completed
-   Story points completed

Note:

Quantity alone does not always represent quality. A developer producing
thousands of lines of code may simply be producing thousands of future
bugs. Software has a talent for making simple things complicated.

------------------------------------------------------------------------

# Common Productivity Metrics

## Velocity

Measures work completed in a sprint.

Example:

    Sprint 1:

    40 Story Points

------------------------------------------------------------------------

## Throughput

Measures completed work over time.

Example:

    10 Features per month

------------------------------------------------------------------------

# 5. Risk Metrics

Measure project uncertainty.

Examples:

-   Number of identified risks
-   High-impact risks
-   Risk resolution rate

------------------------------------------------------------------------

# 6. Agile Metrics

Agile teams use specific metrics.

------------------------------------------------------------------------

# Velocity

Amount of work completed in a sprint.

Example:

    Sprint 1 = 30 points

    Sprint 2 = 35 points

------------------------------------------------------------------------

# Burndown Chart

Shows remaining work during a sprint.

Example:

    Remaining Work

    |

    |

    |\
    | \
    |  \
    |___\________

         Time

------------------------------------------------------------------------

# Burnup Chart

Shows completed work over time.

Tracks:

-   Completed work
-   Total scope

------------------------------------------------------------------------

# Lead Time

Time from request creation to delivery.

Example:

    Feature Request

            ↓

    Delivered Feature

------------------------------------------------------------------------

# Cycle Time

Time taken to complete a task after work starts.

------------------------------------------------------------------------

# 7. Earned Value Metrics

Earned Value Management measures project performance.

Important values:

------------------------------------------------------------------------

# Planned Value (PV)

Amount of planned work.

------------------------------------------------------------------------

# Earned Value (EV)

Value of completed work.

------------------------------------------------------------------------

# Actual Cost (AC)

Actual money spent.

------------------------------------------------------------------------

# Performance Index Metrics

## Schedule Performance Index (SPI)

Formula:

    SPI = EV / PV

Meaning:

    SPI > 1

    Ahead of schedule


    SPI < 1

    Behind schedule

------------------------------------------------------------------------

## Cost Performance Index (CPI)

Formula:

    CPI = EV / AC

Meaning:

    CPI > 1

    Under budget


    CPI < 1

    Over budget
