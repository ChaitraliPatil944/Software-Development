# Work Breakdown Structure (WBS)

WBS divides a project into smaller tasks.

Example:

``` text
Software Project

├── Requirements

├── Design

├── Development

├── Testing

└── Deployment
```

Benefits:

-   Better estimation
-   Clear responsibilities
-   Easier tracking

# Critical Path Method (CPM)

CPM identifies the longest sequence of dependent tasks.

Critical path:

-   Determines project duration
-   Has zero float

------------------------------------------------------------------------

# CPM Concepts

## Earliest Start (ES)

Earliest beginning time.

## Earliest Finish (EF)

    EF = ES + Duration

## Latest Start (LS)

Latest allowed start.

## Latest Finish (LF)

Latest allowed finish.

------------------------------------------------------------------------

# Float

Amount of delay possible without affecting project completion.

Formula:

    Float = LS - ES
