# What is Logging?

**Logging** is the process of recording events and activities generated
by applications, systems, and infrastructure.

Logs help teams understand:

-   What happened?
-   When did it happen?
-   Why did it happen?

# Types of Logs

Common log types:

1.  Application Logs
2.  System Logs
3.  Security Logs
4.  Database Logs
5.  Network Logs

------------------------------------------------------------------------

# 1. Application Logs

Generated by applications.

Examples:

-   User login
-   API requests
-   Application errors

Example:

    User authentication failed

------------------------------------------------------------------------

# 2. System Logs

Generated by operating systems.

Examples:

-   Server startup
-   Hardware events
-   System failures

------------------------------------------------------------------------

# 3. Security Logs

Track security-related activities.

Examples:

-   Login attempts
-   Permission changes
-   Suspicious activity

------------------------------------------------------------------------

# 4. Database Logs

Record database activities.

Examples:

-   Queries
-   Transactions
-   Errors

------------------------------------------------------------------------

# 5. Network Logs

Track network behavior.

Examples:

-   Connections
-   Traffic
-   Firewall events

------------------------------------------------------------------------

# Log Levels

Logs are categorized by importance.

Common levels:

    DEBUG

    INFO

    WARNING

    ERROR

    CRITICAL

------------------------------------------------------------------------

# 1. DEBUG

Detailed information used during development.

Example:

    Variable value = 100

------------------------------------------------------------------------

# 2. INFO

Normal application events.

Example:

    User logged in successfully

------------------------------------------------------------------------

# 3. WARNING

Potential problem.

Example:

    Memory usage increasing

------------------------------------------------------------------------

# 4. ERROR

Application failure.

Example:

    Database connection failed

------------------------------------------------------------------------

# 5. CRITICAL

Severe issue requiring immediate action.

Example:

    Server unavailable

------------------------------------------------------------------------

# Logging vs Monitoring

  Logging                Monitoring
  ---------------------- ----------------------
  Records events         Tracks system health
  Detailed information   Metrics and alerts
  Helps find cause       Detects issues

Example:

Monitoring:

    CPU = 95%

Logging:

    Process crashed at 10:30 PM

# Logging Best Practices

## 1. Use Meaningful Messages

Bad:

    Error

Good:

    Database connection timeout

------------------------------------------------------------------------

## 2. Avoid Sensitive Data

Do not log:

-   Passwords
-   Tokens
-   Private information

------------------------------------------------------------------------

## 3. Use Proper Log Levels

Do not mark everything as ERROR.

------------------------------------------------------------------------

## 4. Centralize Logs

Use centralized platforms.

------------------------------------------------------------------------

## 5. Rotate Logs

Prevent unlimited storage growth.

# Summary

Logging is a critical DevOps practice that provides detailed information
about system behavior. By collecting, storing, and analyzing logs, teams
can troubleshoot issues faster, improve security, and maintain reliable
applications.

Monitoring tells teams that something is wrong, while logging helps
explain why it happened.
