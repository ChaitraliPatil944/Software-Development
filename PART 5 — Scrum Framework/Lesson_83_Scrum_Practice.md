A Scrum professional should be able to:

- Create backlog items
- Plan Sprints
- Estimate work
- Handle blockers

# Section 2 — Create User Stories

## Exercise

Create User Stories for an E-Commerce Application.

Format:

```text
As a <user>
I want <feature>
So that <benefit>
```

---

## Example Answers

### Login

```
As a customer,
I want to login,
so that I can access my account.
```

---

### Order Tracking

```
As a customer,
I want to track my order,
so that I know delivery status.
```

---

### Admin Management

```
As an admin,
I want to manage products,
so that I can maintain inventory.
```

# Section 9 — Sprint Review Practice

## Scenario

A team completes checkout functionality.

Stakeholder says:

"We also need coupon support."

What happens?

Answer:

Product Owner adds and prioritizes the requirement in Product Backlog.

# Section 10 — Retrospective Practice

## Problem

Sprint delays happened because testing started late.

Find improvement.

Example:

Problem:

Late testing.

Root Cause:

Testing was not included early.

Action:

Include QA activities during Sprint planning.
