# 1. SDLC at a Glance

**Software Development Life Cycle (SDLC)** is a structured process used to build, deliver, and maintain software.

Goals:

- Deliver quality software
- Reduce risk
- Improve planning
- Control cost
- Increase customer satisfaction

---

# 2. Complete SDLC Flow

```text
Business Idea
      ↓
Requirement Gathering
      ↓
Requirement Analysis
      ↓
Feasibility Study
      ↓
SRS
      ↓
Design
      ↓
Development
      ↓
Testing
      ↓
Deployment
      ↓
Maintenance
```

---

# 3. SDLC Phases Summary

| Phase | Purpose | Main Output |
|-------|---------|-------------|
| Requirement Gathering | Collect needs | Requirement List |
| Requirement Analysis | Validate & refine | Approved Requirements |
| Feasibility Study | Check viability | Feasibility Report |
| SRS | Document requirements | SRS |
| Design | Technical blueprint | HLD, LLD, UML |
| Development | Build software | Source Code |
| Testing | Verify quality | Test Reports |
| Deployment | Release software | Live Application |
| Maintenance | Improve software | Updates & Patches |

# 5. Functional vs Non-Functional Requirements

| Functional | Non-Functional |
|------------|----------------|
| What the system does | How well it performs |
| Login | Login < 2 sec |
| Payment | Secure payment |
| Search | Fast search |
