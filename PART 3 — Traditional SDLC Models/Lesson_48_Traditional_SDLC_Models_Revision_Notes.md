# 1. Traditional Models Overview

Traditional SDLC models provide structured approaches for planning, building, testing, deploying, and maintaining software.

Models covered:

- Waterfall
- V-Model
- Incremental
- Iterative
- Spiral
- RAD
- Prototype
- Big Bang
- Component-Based Development (CBD)

# 3. One-Line Definitions

| Model | One-Line Definition |
|------|----------------------|
| Waterfall | Sequential development with fixed phases |
| V-Model | Waterfall with corresponding testing phases |
| Incremental | Deliver software feature by feature |
| Iterative | Improve the product repeatedly |
| Spiral | Risk-driven iterative development |
| RAD | Rapid development using prototypes and reuse |
| Prototype | Build a model to clarify requirements |
| Big Bang | Minimal planning and maximum experimentation |
| CBD | Assemble software using reusable components |

---

# 4. Best Use Cases

| Model | Best For |
|------|----------|
| Waterfall | Stable, regulated projects |
| V-Model | Safety-critical systems |
| Incremental | Large business applications |
| Iterative | Evolving products |
| Spiral | High-risk projects |
| RAD | Fast business applications |
| Prototype | Unclear requirements |
| Big Bang | Research & PoC |
| CBD | Enterprise & cloud systems |

---

# 5. Strengths

- Waterfall → Documentation
- V-Model → Verification & Validation
- Incremental → Early delivery
- Iterative → Continuous improvement
- Spiral → Risk management
- RAD → Speed
- Prototype → Requirement clarity
- Big Bang → Innovation
- CBD → Reusability

---

# 6. Limitations

- Waterfall → Difficult requirement changes
- V-Model → Heavy planning
- Incremental → Integration effort
- Iterative → Scope growth
- Spiral → Costly & complex
- RAD → Customer dependency
- Prototype → Scope creep
- Big Bang → High uncertainty
- CBD → Third-party dependency

---

# 12. Exam Tips

- Define the model first.
- Explain why it exists.
- Mention strengths and limitations.
- State best use cases.
- Compare it with another model.
- Draw the workflow whenever possible.
