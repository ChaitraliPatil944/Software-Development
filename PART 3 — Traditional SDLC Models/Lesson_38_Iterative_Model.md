# Introduction

Imagine developing an online shopping application.

Instead of building every feature perfectly on the first attempt, the team creates a simple version, gathers feedback, improves it, and repeats the cycle.

```text
Version 1
     ↓
Feedback
     ↓
Improve
     ↓
Version 2
     ↓
Feedback
     ↓
Improve
     ↓
Version 3
```

This continuous refinement is known as the **Iterative Model**.

---

# What is the Iterative Model?

The **Iterative Model** develops software through multiple repeated cycles called **iterations**.

Each iteration produces a better version of the software.

Unlike traditional models, improvement happens continuously.

---

# Why Was the Iterative Model Introduced?

Many software projects have:

- Changing business needs
- New customer feedback
- Improved technical understanding
- New market demands

Instead of assuming the first design is perfect, the Iterative Model embraces continuous improvement.

---

# Industry Example

## AI Image Editing Application

### Iteration 1

- Upload Image
- Basic Filters

### Feedback

Users request:

- Faster processing
- Better UI
- More filters

### Iteration 2

- Performance improvements
- New interface
- Additional filters

### Iteration 3

- AI enhancements
- Batch processing
- Cloud storage

The application becomes more capable after every iteration.

---

# Activities in Each Iteration

1. Analyze feedback
2. Update requirements
3. Design improvements
4. Development
5. Testing
6. Release
7. Gather feedback

The cycle repeats until project goals are achieved.

---

# Advantages

- Early working software
- Continuous customer involvement
- Easier requirement changes
- Better product quality
- Lower project risk
- Frequent validation

---

# Limitations

- Requires active customer participation
- Scope may grow over time
- Planning becomes more difficult
- Documentation can be harder to maintain

---

# Best Use Cases

Ideal for:

- AI products
- Research projects
- Startup applications
- User-centric products
- Innovative software

---

# Iterative vs Incremental

| Iterative | Incremental |
|-----------|-------------|
| Improve existing software | Add new features |
| Same functionality becomes better | Functionality grows over time |
| Focus on refinement | Focus on expansion |

---

# Iterative vs Waterfall

| Waterfall | Iterative |
|-----------|-----------|
| One pass | Multiple cycles |
| Fixed design | Continuous improvement |
| Late feedback | Early and repeated feedback |

# Common Mistakes

❌ Ignoring customer feedback.

✔ Every iteration should incorporate learning.

❌ Making uncontrolled changes.

✔ Prioritize improvements carefully.

❌ Skipping testing between iterations.

✔ Validate every iteration before release.

# Summary

The Iterative Model develops software through repeated cycles of design, development, testing, and feedback. Instead of expecting perfection in the first release, it continuously improves the product based on stakeholder input and technical learning. This approach reduces risk, supports changing requirements, and produces higher-quality software over time.
