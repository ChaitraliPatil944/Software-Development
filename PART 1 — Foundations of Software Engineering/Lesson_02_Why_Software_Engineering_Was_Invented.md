# Lesson 2 — Why Software Engineering Was Invented

# Part 1 — Foundations of Software Engineering

> **Objective**
>
> Understand **why Software Engineering came into existence**, what problems existed before it, and why modern software development cannot rely on programming alone.

---

# Imagine the World Before Software Engineering

In the early days of computing (1950s–1960s), software was small.

One programmer often wrote an entire application.

Example:

```
Calculator
  |
One Programmer
  |
Finished
```

There were:
- Few users
- Small codebases
- Limited hardware
- Short development cycles

As computers became more powerful, software became dramatically larger.

Examples:
- Banking systems
- Airline reservation systems
- Operating systems
- Space missions

Suddenly, one person could no longer understand the whole system.

---

# The Problem

Projects started failing because teams only knew **how to program**, not **how to engineer software**.

Common problems included:

- Projects delivered years late
- Budgets exceeded
- Software crashed frequently
- Requirements kept changing
- Poor documentation
- Difficult maintenance

Programming answered:

> "How do I write this code?"

Software Engineering answered:

> "How do we build reliable software that hundreds of people can develop together?"

---

# The Software Crisis

During the late 1960s, the industry experienced what became known as the **Software Crisis**.

Symptoms included:

- Cost overruns
- Missed deadlines
- Low quality
- Difficult maintenance
- Project failures

```
Idea
 |
Development
 |
More Features
 |
More Complexity
 |
Bugs Increase
 |
Delays
 |
Higher Cost
 |
Customer Dissatisfied
```

This crisis led researchers and companies to search for disciplined development methods.

---

# Why Programming Alone Was Not Enough

Consider constructing a hospital.

You would never ask one mason to design electrical wiring, plumbing, architecture, safety rules, and interior design.

Software became equally complex.

Large applications require:

- Planning
- Design
- Architecture
- Documentation
- Team collaboration
- Testing
- Maintenance
- Project management

Coding is only one activity among many.

---

# Birth of Software Engineering

The term **Software Engineering** gained prominence after the 1968 NATO Conference on Software Engineering.

Its goal was to bring engineering principles into software development.

Engineering means:

- Systematic
- Predictable
- Repeatable
- Measurable
- Maintainable

---

# Real-Life Analogy

Imagine preparing dinner.

Without Software Engineering:

```
Start Cooking
 |
Forgot Ingredients
 |
No Recipe
 |
Food Burns
 |
Start Again
```

With Software Engineering:

```
Plan Menu
 |
Buy Ingredients
 |
Prepare
 |
Cook
 |
Taste
 |
Serve
```

Planning reduces mistakes.

---

# Industry Example

Imagine building an online shopping platform.

Without Software Engineering:

- Developers start coding immediately.
- No requirements are documented.
- Different teams build incompatible features.
- Bugs multiply.
- Deadlines slip.

With Software Engineering:

```
Requirements
      |
Architecture
      |
Development
      |
Testing
      |
Deployment
      |
Maintenance
```

Every stage has defined responsibilities.

---

# Why It Is Necessary Today

Modern applications may involve:

- Millions of users
- Thousands of developers
- Hundreds of microservices
- Multiple countries
- Continuous updates

Examples include:

- E-commerce
- Banking
- Healthcare
- Streaming
- Social media

Such systems require disciplined processes.

---

# Benefits of Software Engineering

- Better quality
- Lower development cost over time
- Easier maintenance
- Predictable schedules
- Better teamwork
- Improved customer satisfaction
- Higher reliability
- Better scalability

---

# Programming vs Software Engineering

| Programming | Software Engineering |
|-------------|----------------------|
| Writes code | Builds complete software |
| Individual focus | Team focus |
| Solves one problem | Solves entire product lifecycle |
| Short-term | Long-term |
| Limited documentation | Extensive documentation |
| Few processes | Defined processes |

---

# Key Takeaways

Software Engineering was invented because software became too large and complex to manage through programming alone.

It introduced structured processes that help teams deliver reliable, maintainable, and scalable software.

# Summary

Software Engineering emerged because software systems outgrew the capabilities of ad hoc programming. By applying engineering principles such as planning, design, testing, documentation, and maintenance, organizations can build software that is reliable, scalable, and easier to evolve throughout its lifecycle.
