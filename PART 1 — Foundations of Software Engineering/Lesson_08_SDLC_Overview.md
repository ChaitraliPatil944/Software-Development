# Introduction

Imagine someone asks you to build a hospital.

Would you immediately start laying bricks?

Of course not.

You would first:

- Understand the requirements
- Create a blueprint
- Estimate cost
- Build
- Inspect
- Maintain

Software follows the same philosophy.

That structured approach is called the **Software Development Life Cycle (SDLC).**

---

# What is SDLC?

**Software Development Life Cycle (SDLC)** is a structured process used to plan, design, develop, test, deploy, and maintain software efficiently and systematically.

Its purpose is to produce high-quality software while minimizing cost, risk, and development time.

---

# Why Was SDLC Needed?

Before SDLC, teams often started coding immediately.

Result:

```text
 |

 |
More Bugs
 |

 |

 |

```

SDLC introduced planning and discipline.

```text
Idea
 |

 |

 |

 |

 |

 |
```

# Phase 1: Requirements

Questions answered:

- What should we build?
- Who are the users?
- What problems are we solving?

Output:

- Requirement documents
- User stories
- Initial scope

---

# Phase 2: Design

Architects prepare the blueprint.

Includes:

- System architecture
- Database design
- UI design
- API design

Output:

- Design documents

---

# Phase 3: Development

Developers convert designs into working software.

Activities:

- Coding
- Unit testing
- Code reviews

Output:

- Working application

---

# Phase 4: Testing

Quality engineers verify the software.

Checks include:

- Functional testing
- Performance testing
- Security testing
- Regression testing

Output:

- Bug reports
- Stable release candidate

---

# Phase 5: Deployment

Software is delivered to users.

Examples:

- Web server
- Mobile app stores
- Enterprise servers
- Cloud platforms

Output:

- Live software

---

# Phase 6: Maintenance

After release, work continues.

Activities:

- Bug fixes
- Performance improvements
- Security patches
- New features

Maintenance often consumes the largest share of a software product's lifetime.

# Common Misconceptions

❌ SDLC means Waterfall.

✔ Waterfall is only one SDLC model.

❌ SDLC ends after deployment.

✔ Maintenance is an essential SDLC phase.

❌ SDLC is only for large companies.

# Summary

The Software Development Life Cycle (SDLC) provides a systematic framework for transforming an idea into reliable software. By organizing work into well-defined phases, SDLC improves quality, reduces risk, simplifies collaboration, and ensures software can be maintained and enhanced throughout its lifecycle. Every modern development methodology, from Waterfall to Agile, is built upon the core principles of the SDLC.
